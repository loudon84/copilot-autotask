PRD A：AutoTask 前端 v0.6 方案
1. 版本定位

版本：v0.6

主题：Remote First + 客户 SRM 门户管理

v0.6 的目标是把 AutoTask Studio 从“Mock UI 原型”推进到“真实后端数据驱动”的桌面工作台。前端不再默认读取 src/mock，而是默认调用 nodeskclaw-task。nodeskclaw-task 的 Client API 前缀是 /api/v1/autotask，服务端口为 4520，并与 nodeskclaw-backend 平级部署。

2. 核心目标

v0.6 前端需要完成：

模块	v0.6 要求
全局数据模式	默认 remote mode
客户 SRM	使用 PortalAccount 后端接口
新增 SRM	调用 POST /portal-accounts
编辑 SRM	调用 PATCH /portal-accounts/{id}
删除 SRM	调用 DELETE /portal-accounts/{id}
查看详情	调用 GET /portal-accounts/{id}
快速打开	使用后端 Portal 数据 + Electron Web 工作区打开
权限	根据 403 / Portal 权限做 UI 降级
Mock	仅允许作为 dev fallback，默认关闭
3. 非目标

v0.6 客户 SRM 页面不做：

不绑定 RPA Flow
不维护 WorkflowBinding
不维护 locatorProfile
不维护字段映射
不做自动登录
不保存明文密码
不实现真实 RPA 执行

当前前端 SRMPortal 类型里包含 serverRpaProfileId、locatorProfile、fieldMapping、mfaPolicy 等字段，v0.6 页面需要把这些从客户 SRM 维护表单中剥离，不作为用户当前维护项。

5.2 API Response 解包

nodeskclaw-task 返回统一包装体：

{
  "code": 0,
  "error_code": null,
  "message_key": null,
  "message": "success",
  "data": {}
}

并且业务 data 使用 camelCase 字段。

前端必须新增统一解包：

export interface ApiResponse<T> {
  code: number
  errorCode?: number | null
  messageKey?: string | null
  message: string
  data: T
}

export function unwrapApiResponse<T>(response: ApiResponse<T>): T {
  if (response.code !== 0) {
    throw new Error(response.message || '请求失败')
  }
  return response.data
}

apiRequest<T> 不应直接把 JSON 当成业务对象，而应返回 ApiResponse<T> 后解包。

11. 快速打开设计

v0.6 的客户 SRM 只需要快速打开，不绑定 RPA。

打开流程：

用户点击【打开】
  ↓
前端检查 portal.status === ENABLED
  ↓
调用后端 test-open 记录权限与审计
  ↓
Electron Web 工作区打开 portalUrl
  ↓
使用 clientSessionPartition 隔离 Session

接口：

POST /api/v1/autotask/portal-accounts/{id}/test-open

Electron 调用：

window.autotaskWeb.openPortal({
  portalAccountId: portal.id,
  url: portal.portalUrl,
  title: portal.portalName,
  partition: portal.clientSessionPartition,
  openMode: portal.clientOpenMode
})
12. 权限与错误处理

nodeskclaw-task 的 Client API 需要 Bearer JWT，并且服务本身不提供登录接口，客户端沿用 nodeskclaw-backend 签发的 JWT。

前端处理规则：

后端状态	前端动作
401	调用 refreshAuth，失败后跳转登录
403	展示“权限不足”，隐藏新增/编辑/删除
404	展示空状态或“资源不存在”
422	表单字段级提示
500	toast + 保留当前页面

当前前端 apiRequest 已经有 401 refresh、403 toast、500 toast 的基础逻辑。
