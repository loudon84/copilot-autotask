# AutoTask / nodeskclaw-task 修复 PRD v0.5.1

## 去除 keytar 依赖 + 启动稳定性修复 + TokenStore 安全降级方案

## 1. 版本信息

```text
版本：v0.5.1
类型：Hotfix / Architecture Correction
范围：AutoTask Client + nodeskclaw-task
优先级：P0
目标：修复因 keytar native module 引入导致的 Electron 启动失败，并修正 nodeskclaw-task 配置解析问题
```

## 2. 背景

在 v0.5 方案中，为了复用 copilot-desktop / ai-os-desktop 的桌面端登录模式，PRD 中建议使用 `keytar` 作为 Token 安全存储方案。

该设计在生产安全性上有合理性，但对当前 AutoTask 阶段不合适。

当前阶段的真实目标是：

```text
1. AutoTask 从 mock 数据切换到 nodeskclaw-task API
2. AutoTask 登录接入 nodeskclaw-backend
3. AutoTask Renderer 不直接长期持有 token
4. Electron Client 可稳定开发、启动、打包
5. nodeskclaw-task 可稳定启动，支持真实 API 联调
```

引入 `keytar` 后出现 Electron 主进程构建失败：

```text
[UNLOADABLE_DEPENDENCY] Could not load node_modules/keytar/build/Release/keytar.node
stream did not contain valid UTF-8
```

该问题本质是：`keytar` 是 native addon，包含 `.node` 二进制文件。Vite / Electron 主进程构建链路在未正确 externalize / unpack / rebuild native module 时，会错误处理该二进制文件，导致启动失败。

因此 v0.5.1 对 v0.5 做架构纠偏。

## 3. v0.5.1 核心结论

### 3.1 MVP 阶段禁止引入 keytar

```text
v0.5.1 明确规定：
AutoTask MVP 阶段不引入 keytar。
AutoTask MVP 阶段不引入任何 native credential addon。
AutoTask MVP 阶段不要求接系统钥匙串。
```

### 3.2 使用 Electron safeStorage + userData 加密文件

v0.5.1 的 TokenStore 改为：

```text
主方案：
  Electron safeStorage + app.getPath('userData') 加密文件

降级方案：
  safeStorage 不可用时，使用 memory session

禁止方案：

  明文 localStorage
  Renderer 持久化 accessToken
```

生产版本后续可评估：

```text

2. OS Credential Manager
3. 企业密钥服务
4. 硬件安全模块
5. SSO + 短会话 token
```

但不进入 v0.5.1。

## 8. TokenStore 新设计

### 8.1 设计原则

```text
1. token 存储只发生在 Electron Main Process
2. Renderer 不持久化 accessToken
3. Renderer 只保存 public auth state，例如 user、isAuthenticated、expiresAt
4. token 文件使用 safeStorage 加密
5. safeStorage 不可用时只使用内存 session
6. TokenStore 保持接口稳定，后续可替换为 keytar / OS credential
```

### 8.2 TokenStore 状态模型

```ts
export interface StoredAuthSession {
  accessToken: string
  refreshToken?: string
  tokenType: string
  expiresAt?: number
  user?: {
    id: string
    email?: string
    name?: string
    roles?: string[]
    orgId?: string
  }
}
```

### 8.3 TokenStore 文件路径

```text
app.getPath('userData')/autotask-auth-session.bin
```

不允许放在项目目录、renderer public 目录、localStorage、sessionStorage。

### 8.4 TokenStore 实现

```ts
import { app, safeStorage } from 'electron'
import fs from 'node:fs/promises'
import path from 'node:path'

export interface StoredAuthSession {
  accessToken: string
  refreshToken?: string
  tokenType: string
  expiresAt?: number
  user?: {
    id: string
    email?: string
    name?: string
    roles?: string[]
    orgId?: string
  }
}

const TOKEN_FILE = 'autotask-auth-session.bin'

export class TokenStore {
  private memorySession: StoredAuthSession | null = null

  private getTokenFilePath() {
    return path.join(app.getPath('userData'), TOKEN_FILE)
  }

  async get(): Promise<StoredAuthSession | null> {
    if (this.memorySession) {
      return this.memorySession
    }

    if (!safeStorage.isEncryptionAvailable()) {
      return null
    }

    try {
      const encrypted = await fs.readFile(this.getTokenFilePath())
      const decrypted = safeStorage.decryptString(encrypted)
      const session = JSON.parse(decrypted) as StoredAuthSession
      this.memorySession = session
      return session
    } catch {
      return null
    }
  }

  async set(session: StoredAuthSession): Promise<void> {
    this.memorySession = session

    if (!safeStorage.isEncryptionAvailable()) {
      return
    }

    const filePath = this.getTokenFilePath()
    const encrypted = safeStorage.encryptString(JSON.stringify(session))

    await fs.mkdir(path.dirname(filePath), { recursive: true })
    await fs.writeFile(filePath, encrypted)
  }

  async clear(): Promise<void> {
    this.memorySession = null

    try {
      await fs.rm(this.getTokenFilePath(), { force: true })
    } catch {
      // ignore
    }
  }

  async hasSession(): Promise<boolean> {
    const session = await this.get()
    return Boolean(session?.accessToken)
  }
}

export const tokenStore = new TokenStore()
```

## 10. API Request IPC 设计

### 10.1 Renderer 调用

Renderer 不直接调 `fetch(nodeskclaw-task)`，而是：

```ts
window.autotaskApi.request({
  method: 'GET',
  path: '/tasks',
  query: {
    status: 'READY',
  },
})
```

### 10.2 Main Process 执行

Main Process 负责：

```text
1. 读取 endpoint config
2. 读取 tokenStore
3. 注入 Authorization: Bearer <accessToken>
4. 请求 nodeskclaw-task
5. 遇到 401 时 refresh
6. refresh 成功后重放请求
7. refresh 失败后清理 tokenStore
8. 返回 JSON 给 Renderer
```

### 10.3 Renderer 暴露接口

```ts
declare global {
  interface Window {
    autotaskApi: {
      request: <T = unknown>(input: ApiRequestInput) => Promise<T>
    }
  }
}
```

# Part B：nodeskclaw-task 配置修复方案

## 15. 问题背景

nodeskclaw-task 启动时报错：

```text
SettingsError: error parsing value for field "CORS_ORIGINS" from source "DotEnvSettingsSource"
```

原因：

```text
CORS_ORIGINS 在 Settings 中定义为 list[str]
pydantic-settings 默认将 list[str] 环境变量当作 JSON 字符串解析
.env 中使用了逗号分隔格式，导致 json.loads() 失败
```

---

## 16. nodeskclaw-task 修复目标

```text
1. Settings 初始化不因 CORS_ORIGINS 格式失败
2. 支持 JSON array 格式
3. 可选支持逗号分隔格式
4. .env.example 给出正确示例
5. 启动失败时错误信息更清晰
```

## 17. 推荐修复方案

### 17.1 使用 NoDecode + validator

`app/core/config.py`：

```python
from typing import Annotated

from pydantic import field_validator
from pydantic_settings import BaseSettings, NoDecode, SettingsConfigDict


class Settings(BaseSettings):
    APP_NAME: str = "nodeskclaw-task"
    APP_ENV: str = "development"

    HOST: str = "0.0.0.0"
    PORT: int = 4520

    DATABASE_URL: str

    JWT_SECRET: str
    JWT_ALGORITHM: str = "HS256"

    NODESKCLAW_BACKEND_URL: str = "http://127.0.0.1:4510"
    NODESKCLAW_AUTH_ME_PATH: str = "/api/v1/auth/me"

    CORS_ORIGINS: Annotated[list[str], NoDecode] = [
        "http://127.0.0.1:5173",
        "http://localhost:5173",
        "http://127.0.0.1:5174",
        "http://localhost:5174",
        "http://127.0.0.1:4517",
        "http://localhost:4517",
    ]

    @field_validator("CORS_ORIGINS", mode="before")
    @classmethod
    def parse_cors_origins(cls, value):
        if value is None:
            return []

        if isinstance(value, list):
            return value

        if isinstance(value, str):
            value = value.strip()

            if not value:
                return []

            if value.startswith("["):
                import json
                parsed = json.loads(value)
                if not isinstance(parsed, list):
                    raise ValueError("CORS_ORIGINS JSON value must be a list")
                return parsed

            return [item.strip() for item in value.split(",") if item.strip()]

        return value

    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        extra="ignore",
        case_sensitive=True,
    )


settings = Settings()
```

# Part E：风险与规避

## 25. 风险 1：safeStorage 在部分 Linux 环境不可用

规避：

```text
1. safeStorage.isEncryptionAvailable() 检查
2. 不可用时使用 memory session
3. 开发环境允许重新登录
4. 生产环境后续单独做凭证安全专项
```

## 26. 风险 2：Renderer 需要 token 调试

规避：

```text
1. 不暴露 getAccessToken
2. 调试通过 Main Process 日志排查
3. 如必须暴露，使用 dev-only flag，默认关闭
```

## 27. 风险 3：CORS 配置过宽

规避：

```text
1. 开发环境允许 localhost
2. 生产环境明确配置固定域名
3. 不使用 "*" + allow_credentials=True
```

## 28. 风险 4：Token refresh 并发

规避：

```text
1. Main Process 内部维护 refreshPromise
2. 多个 401 请求复用同一个 refresh
3. refresh 成功后统一重放请求
4. refresh 失败后统一 logout
```
