# AutoTask Studio PRD v0.4

## Electron WebContentsView Web 工作台 + 类型 A 人工介入方案

版本：v0.4
定位：AutoTask Studio Desktop 从“本地 RPA 执行客户端”调整为“统一业务操作工作台 + 客户 SRM 管理入口 + Web 操作容器”。
核心变化：浏览器打开方案从 Playwright/CDP 改为 **Electron WebContentsView**。人工介入仅支持 **类型 A：用户自己完成业务动作，不要求服务器继续同一浏览器会话**。

---

## 1. 背景与调整目标

当前 AutoTask Studio 已完成 Mock UI 原型，已有工作台、自动化任务、流程模板、RPA 组件库、客户 SRM、运行监控、证据中心、系统设置等产品模块。原型阶段的数据通过本地 JSON mock，后续可替换为真实 API。

v0.4 的目标是调整浏览器与人工介入方案：

```text
1. Desktop 不再承担 Playwright/CDP 自动化浏览器接管。
2. Desktop 使用 Electron WebContentsView 作为内置 Web 工作区。
3. 用户在客户 SRM 页面中的人工动作，由本地 WebContentsView 打开页面完成。
4. 用户完成后在任务中点击【确认已完成】。
5. 服务器不需要继续使用同一个浏览器会话。
6. Playwright/CDP 只保留在服务器 RPA Runtime 内部。
```

---

## 2. v0.4 产品定位

AutoTask Studio Desktop 是用户的统一操作工作台：

```text
管理客户 SRM 信息
查看自动化任务
查看运行记录
查看待人工任务
打开客户 SRM 门户
打开普通业务 Web 页面
在内置 Web 工作区中完成人工操作
确认人工操作完成
查看运行日志与证据
```

Server RPA Runtime 是后台自动化执行器：

```text
执行 Playwright RPA
登录 SRM
下载文件
上传文件
截图
Trace
生成运行事件
进入 WAITING_HUMAN
等待用户确认人工完成
```

Desktop 和 Server RPA 浏览器会话 **不共享**。

## 3. v0.4 不做事项

```text
不在 Desktop 侧运行 Playwright
不在 Desktop 侧打开 CDP 调试端口
不通过 Desktop 接管服务器浏览器
不做 noVNC / WebRTC 远程浏览器画面
不要求用户操作后服务器继续同一浏览器上下文
不把 Electron WebContentsView 登录态同步给服务器 Playwright
不在 Desktop 保存真实明文密码
不做完整浏览器自动化录制器
```

## 5. 核心产品能力

### 5.1 客户 SRM 快速打开

用户在“客户 SRM”列表点击【快速打开】：

```text
1. Desktop 根据 portalId 获取 SRM 门户配置。
2. Electron Main 创建或激活对应 WebContentsView Tab。
3. WebContentsView 加载 portal.url。
4. WebContentsView 使用 portalId 专属 session partition。
5. 用户可在内置 Web 工作区中登录和操作。
```

展示位置：

```text
客户 SRM 列表
客户 SRM 详情
任务详情右侧操作区
全局搜索 Command
Web 工作区 Tab 列表
```

### 5.2 待人工任务快速打开

任务状态为 `WAITING_HUMAN` 时，用户点击【快速打开】：

```text
1. Desktop 调用 Backend 获取 humanAction。
2. humanAction 中包含 portalId、targetUrl、instruction。
3. Desktop 打开 WebContentsView Tab。
4. 用户在内置 Web 页面中完成业务操作。
5. 用户回到任务详情点击【确认已完成】。
6. Backend 将任务标记为 SUCCESS_MANUAL 或 HUMAN_CONFIRMED。
7. 不触发服务器继续执行同一浏览器会话。
```

### 5.3 普通 Web 操作工作台

新增“Web 工作区”，支持：

```text
打开客户 SRM
打开任务人工处理链接
打开普通业务网址
多 Tab 管理
刷新
后退
前进
复制链接
在系统浏览器打开
关闭 Tab
恢复最近 Tab
```

### 5.4 登录态保存

Desktop WebContentsView 登录态保存范围：

```text
Electron session partition
cookie
localStorage
IndexedDB
cache
```

推荐 partition：

```text
persist:autotask-global
persist:srm:{portalId}
persist:task:{taskId}
```

规则：

```text
客户 SRM 固定使用 persist:srm:{portalId}
普通 Web 使用 persist:autotask-global
临时任务页面可使用 persist:task:{taskId}
```

注意：

```text
Desktop 登录态只服务用户本地 Web 操作。
Server RPA 登录态由 Server RPA Runtime 独立维护。
二者不共享 cookie / localStorage / storageState。
```

## 6. 人工介入定义

### 6.1 支持的人工介入类型

v0.4 只支持类型 A：

```text
类型 A：用户自己完成业务动作，不要求服务器继续同一浏览器会话
```

适用场景：

```text
用户进入客户 SRM 手动确认订单
用户进入客户 SRM 手动上传附件
用户进入客户 SRM 手动完成验证码
用户进入客户 SRM 手动审批提交
用户进入客户 SRM 查看异常并自行处理
```

### 6.2 不支持的人工介入类型

v0.4 不支持类型 B：

```text
服务器 Playwright 已经运行到某个页面
用户接管同一个服务器浏览器
用户处理后服务器继续从下一步执行
```

这类场景后续需要远程浏览器画面方案，例如 Browserless Live Session、noVNC、WebRTC、远程 Chrome DevTools。v0.4 不实现。

## 7. 任务状态机

v0.4 任务状态：

```text
DRAFT
READY
QUEUED
RUNNING
WAITING_HUMAN
HUMAN_OPERATING
HUMAN_CONFIRMED
SUCCESS
SUCCESS_MANUAL
PARTIAL_SUCCESS
FAILED
CANCELLED
```

状态流转：

```text
DRAFT
  ↓
READY
  ↓
QUEUED
  ↓
RUNNING
  ├─ SUCCESS
  ├─ FAILED
  ├─ PARTIAL_SUCCESS
  └─ WAITING_HUMAN
        ↓ 用户点击【快速打开】
      HUMAN_OPERATING
        ↓ 用户点击【确认已完成】
      SUCCESS_MANUAL
```

关键规则：

```text
WAITING_HUMAN 不再自动 resume。
HUMAN_OPERATING 只表示用户已打开人工处理页面。
SUCCESS_MANUAL 表示用户确认已人工完成。
确认后不调用服务器 Playwright 继续执行。
```

## 12. Server RPA Runtime 规则

Server RPA 仍使用 Playwright/CDP，但只在服务器内部使用。

原有规划中 Playwright RPA Worker 负责执行 workflow steps、打开浏览器、登录 Mock SRM、搜索 PO、下载文件、截图、保存 trace、遇到异常上报等能力。

v0.4 调整规则：

```text
1. Server RPA 可以将任务置为 WAITING_HUMAN。
2. Server RPA 生成 HumanAction。
3. Server RPA 不等待用户接管同一个浏览器。
4. 用户确认人工完成后，任务直接进入 SUCCESS_MANUAL。
5. 若后续还需自动化处理，应创建新的独立任务，而不是 resume 原浏览器会话。
```

## 13. 安全策略

### 13.1 Electron 安全边界

```text
Renderer 不直接访问 Node API
Renderer 不直接创建 WebContentsView
Renderer 不直接访问 session / cookie
所有能力通过 IPC 暴露
contextIsolation 必须保持开启
```

### 13.2 WebContentsView 安全策略

```text
禁止任意页面访问 preload API
不向客户 SRM 注入业务脚本
默认不开放 DevTools
下载文件进入用户确认路径或受控目录
权限请求必须弹出确认
外部协议使用 shell.openExternal
```

### 13.3 Session 策略

```text
客户 SRM 独立 session partition
普通 Web 独立 global partition
临时任务可使用 task partition
支持清理单个 portal 的登录态
支持清理所有 Web 工作区缓存
```

### 13.4 URL 白名单

```text
SRMPortal.url 默认可信
HumanAction.targetUrl 必须由后端生成
普通手动 URL 可打开，但需标记 sourceType=manual_url
高风险协议禁止打开：file://、javascript:、data:
```

## 19. v0.4 关键结论

```text
Desktop 的浏览器能力 = Electron WebContentsView Web 工作区
Server 的自动化能力 = Playwright/CDP RPA Runtime
人工介入 = 用户自己完成业务动作 + 手动确认完成
Desktop 与 Server 不共享同一浏览器会话
确认人工完成后任务闭环，不自动 resume 原 RPA 会话
```

这版方案更符合 AutoTask Studio 的新定位：**客户端是统一工作台，服务器是自动化执行器。**
